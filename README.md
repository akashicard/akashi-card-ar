# 明石名物名所AR エンジン（自作WebARプロトタイプ）

GPS判定→カメラ起動→名所名・説明文の重ね合わせ→証拠画像の撮影→サーバーへの送信・クリア数記録まで行う、フロントエンド(`index.html`)＋バックエンド(Google Apps Script)の構成です。LESSARやpalanARのような外部サービスには登録せず、Googleアカウントの無料機能だけで完結します。

## 仕組み（29ヶ所を回って中央窓口で一括引換する運用）

1. 初回アクセス時、端末ごとに6桁の「整理番号」が自動発行され、スマホに保存される（参加者はこの番号を控えておく）
2. 利用者がGoogleマイマップのピンのメモ欄から `index.html?spot=スポットID` のURL（またはQRコード）を開く
3. 「現在地を確認してARを起動」ボタンを押すと、現在地と登録座標との距離を計算
4. 設定した半径以内なら、その場でカメラが起動し、そのスポットの名前と説明文が縁取り文字（背景透過）で重ねて表示される
5. 「撮影する」を押すと、カメラ映像＋名所名・説明文＋スポット名／日時／GPS座標を焼き込んだ画像が生成される（JPEG圧縮で1枚あたり数百KB程度に抑えている）
6. 「送信する」を押すと、画像と整理番号がバックエンド(Apps Script)に送られ、Drive保存＋スプレッドシートに記録される。送信完了後、現在のクリア数（例: 12/29）が画面に表示される
7. 29ヶ所すべて記録されると「コンプリート」表示が出る
8. 利用者は29ヶ所を回り終えたら、1つの中央配布場所へ行き、整理番号をスタッフに伝える。スタッフはスプレッドシートをその番号で検索し、クリア状況を確認してカードを手渡す（引換済みの行にはチェックを入れて二重引換を防止）

GPS判定によるアクセス制限と、画像へのGPS/日時の焼き込みの二重チェックで、現地での撮影であることを確認します。

## スポットの追加・編集方法

`index.html` 内の `SPOTS` という箇所を編集します。

```js
var SPOTS = {
  akashi_castle: {
    name: "明石城",
    lat: 34.6464,
    lng: 134.9990,
    radius: 100,
    description: "1619年に築かれた明石のシンボル。天守台からの眺めが見どころです。"
  },
  ...
};
```

- `akashi_castle` の部分がURLで使うスポットID（半角英数字）
- `lat` / `lng` は現地の正確な緯度経度（**現在入っている値はサンプルの目安値です。公開前に必ず現地でスマートフォンのGPSを使って正確な座標に差し替えてください**）
- `radius` はAR起動を許可する半径（メートル）。スマホのGPS誤差は屋外で10～30m程度あるため、50～150m程度を目安に調整してください
- `description` はAR画面・撮影画像に表示する説明文（名所名は `name` が自動的に大きく表示され、`description` はその下に小さめの文字で表示されます。背景は透過で、縁取り文字なので写真のどの背景でも読みやすくなっています）

スポットを増やす場合は、同じ書式でオブジェクトを追加するだけです。29ヶ所すべて追加してください。`description` が未確定のスポットは、仮の文章を入れておき、確定後に差し替えてください。

## バックエンドの設定（必須）

画像送信・クリア数記録を機能させるには、Apps Scriptのセットアップが必要です。手順は `apps-script/README.md` を参照してください。セットアップ後に発行されるウェブアプリのURLを、`index.html` 内の以下の行に貼り付けます。

```js
var SUBMIT_ENDPOINT = "ここにApps ScriptウェブアプリのURLを貼り付け";
```

未設定のままでも、AR起動・撮影・画像のローカル保存（「画像を保存」ボタン）は動作します。「送信する」を押した時にだけ「送信先が設定されていません」という案内が表示されます。

## Googleマイマップへの貼り方

各ピンのメモ欄に、以下のような内容を記載します（URLは公開後のものに置き換えてください）。

```
到着したらこちらをタップ → https://（公開先URL）/index.html?spot=akashi_castle
名所名と説明文が表示されたら撮影し、画像を配布場所のスタッフへご提示ください。
```

カード裏面用にこのURLをQRコード化しておくと現地での導線がスムーズです。

## 公開（ホスティング）方法

カメラ・位置情報のAPIはHTTPS（またはlocalhost）でないと動作しないため、公開先はHTTPS対応のホスティングが必要です。本プロジェクトでは **GitHub Pages** を採用します（無料・HTTPS自動対応）。

手順は `GITHUB_PAGES_SETUP.md` を参照してください。ターミナル操作不要のドラッグ&ドロップ手順を載せています。公開対象は `index.html` と `assets/` フォルダのみで、`apps-script/` フォルダは公開リポジトリにアップロードしないでください。

（Netlify / Vercel / Firebase Hostingも同様の無料ホスティングとして利用可能ですが、今回はGitHub Pagesで運用します）

## 動作確認時の注意

- ローカルで `index.html` を直接ダブルクリックして開くと、`file://` 扱いとなりカメラ/位置情報が動作しない場合があります。確認時は簡易サーバーを立てるか、上記ホスティング先に公開してから動作確認してください。
- iPhone（Safari）は初回アクセス時に位置情報・カメラの許可ダイアログが出ます。許可しないと起動できません。
- 各スポットの `description`（説明文）が未確定の間は仮の文章が表示されます。スポットが確定したら、正式な紹介文に差し替えてください。
- 「整理番号」は端末のブラウザに保存されています。スマホを変えたりブラウザのデータを消すと引き継げないため、初回起動時の画面に表示される番号を紙のしおりなどに書き留めてもらう案内をしておくと安全です。
- 通信に失敗した場合（電波が悪い場所など）は「送信する」ボタンでもう一度送信できます。撮影した画像はその場で消えないので、電波の良い場所に移動してから再送信しても構いません。
